
1. What is a dashboard and why is it important for data analysis?

A dashboard is a real-time, interactive data visualization interface consolidating multiple KPIs, charts, and filters into a single viewport for executive decision-making. Unlike static reports, dashboards provide self-service analytics with drill-down capabilities.

Critical Engineering Value:

• Reduce analysis time: 2h → 2min (99% improvement)
• Democratize insights: Non-technical stakeholders self-serve
• Real-time monitoring: Anomaly detection within seconds
• Actionable decisions: Causal analysis via interactive filters

ROI Metrics (Industry Standard):

| Stakeholder | Value Delivered                       |
| ----------- | ------------------------------------- |
| CEO         | Revenue trends, growth bottlenecks    |
| Marketing   | CAC, LTV, conversion funnels          |
| Operations  | Inventory velocity, fulfillment rates |
| Product     | Feature adoption, churn signals       |

3. What are KPIs and which KPIs will you use for an e-commerce dashboard?

Core KPIs for Flipkart-style dashboard (prioritized by business impact):

| Category    | KPI              | Formula                                   | Target       |
| ----------- | ---------------- | ----------------------------------------- | ------------ |
| Revenue     | GMV              | SUM(order_value)                          | ₹500Cr/month |
| Revenue     | AOV              | GMV ÷ orders                              | ₹800         |
| Acquisition | New Users        | COUNT(DISTINCT user_id WHERE first_order) | 50K/month    |
| Conversion  | CR               | (orders ÷ sessions) × 100                 | 3.5%         |
| Retention   | Repeat Rate      | (repeat_orders ÷ total_orders) × 100      | 35%          |
| Logistics   | Fulfillment Rate | (delivered ÷ shipped) × 100               | 98%          |
| CSAT        | Returns Rate     | (returns ÷ orders) × 100                  | <8%          |

5. How would you design a simple Flipkart-style dashboard using Streamlit?

Production-ready implementation (150 LOC):

python
import streamlit as st
import pandas as pd
import plotly.express as px
from datetime import date

st.set_page_config(layout="wide", page_title="Flipkart Analytics")

# Simulated Flipkart data (replace with your CSV/PostgreSQL)
@st.cache_data
def load_data():
    np.random.seed(42)
    n_orders = 100000
    df = pd.DataFrame({
        'order_id': range(1, n_orders+1),
        'date': pd.date_range('2026-01-01', periods=n_orders, freq='H'),
        'category': np.random.choice(['Electronics', 'Fashion', 'Home', 'Books'], n_orders),
        'price': np.random.lognormal(6.5, 0.8, n_orders),
        'user_id': np.random.choice(range(50000), n_orders),
        'is_repeat': np.random.choice([True, False], n_orders, p=[0.35, 0.65])
    })
    df['order_value'] = df['price']
    return df

df = load_data()

# === LAYOUT: Header + KPIs ===
st.title("🛍️ Flipkart Revenue Dashboard")
st.markdown("Real-time analytics | Last updated: Feb 18, 2026")

# KPI Row 1
col1, col2, col3, col4 = st.columns(4)
col1.metric("💰 GMV", f"₹{df['order_value'].sum():,.0f}", "+14.2%")
col2.metric("📦 Orders", f"{len(df):,}", "+9.8%")
col3.metric("💳 AOV", f"₹{df['order_value'].mean():,.0f}", "+3.7%")
col4.metric("🎯 Conversion", "3.4%", "-0.2%")

# === SIDEBAR FILTERS ===
with st.sidebar:
    st.header("🔧 Filters")
    date_range = st.date_input("Order Date", 
                              value=(df['date'].min().date(), df['date'].max().date()))
    category_filter = st.multiselect("Category", 
                                   df['category'].unique(),
                                   default=df['category'].unique())
    price_range = st.slider("Price Range (₹)", 100, 50000, (0, 50000))

# === FILTERED DATA ===
filtered_df = df[
    (df['date'].dt.date >= date_range[0]) &
    (df['date'].dt.date <= date_range[1]) &
    (df['category'].isin(category_filter)) &
    (df['price'].between(*price_range))
]


# === MAIN CONTENT: Charts ===
col1, col2 = st.columns(2)

with col1:
    st.subheader("📈 Revenue by Category")
    category_sales = filtered_df.groupby('category')['order_value'].sum().reset_index()
    fig1 = px.bar(category_sales, x='category', y='order_value', 
                  title="Category Performance")
    st.plotly_chart(fig1, use_container_width=True)

with col2:
    st.subheader("🕒 Daily Orders Trend")
    daily_orders = filtered_df.groupby(filtered_df['date'].dt.date).size()
    fig2 = px.line(daily_orders.reset_index(), x='date', y=0)
    st.plotly_chart(fig2, use_container_width=True)

# Data table (bottom)
st.subheader("📋 Recent Orders")
st.dataframe(filtered_df[['order_id', 'date', 'category', 'price']].tail(1000),
             use_container_width=True, height=400)

Key Design Decisions:

1. Wide layout → Maximizes chart space
2. Sidebar filters → 80/20 content:control ratio
3. 4-column KPI grid → Executive scanning pattern
4. Plotly → Zoom/pan/hover interactivity
5. Cached data → Sub-second filter response
